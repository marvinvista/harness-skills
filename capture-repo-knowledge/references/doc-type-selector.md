# Document Type Selector

Choose the smallest durable artifact that preserves the missing context.

Use an ADR when:
- The team chose one technical direction over another
- Future runs will ask "why was this done this way?"

Use a design doc when:
- The change spans multiple layers or systems
- You need context, constraints, rollout notes, and implementation shape

Use a product spec when:
- The missing context is user-facing behavior, acceptance criteria, or workflow intent

Use a reference note when:
- The material is lookup-oriented and not a single decision
- The agent mostly needs a stable pointer and summary
