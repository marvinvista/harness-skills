# Active Vs Completed Plans

Use `active/` when:
- Work is still in flight
- Acceptance criteria are not yet met
- Open risks or next actions remain

Use `completed/` when:
- The planned scope is done
- Follow-up work is either finished or intentionally split into a new plan
- The final state is legible from the plan, code, and logs

Prefer one durable plan per meaningful initiative instead of scattering many tiny plan files.
