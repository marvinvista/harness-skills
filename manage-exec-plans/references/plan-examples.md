# Plan Examples

## Example Request

"Create an execution plan for migrating auth tokens from local storage to HTTP-only cookies."

Expected shape:
- Goal focuses on security and rollout outcome.
- Scope names touched layers or services.
- Acceptance criteria define browser, API, and migration behavior.
- Milestones separate schema changes, rollout, and cleanup.

## Example Request

"Update the search plan after the latest review."

Expected action:
- Refresh progress log with what changed.
- Append new decisions instead of rewriting old ones.
- Narrow next actions to the immediate blocking tasks.

## Example Request

"Close the onboarding plan."

Expected action:
- Confirm the acceptance criteria are met.
- Move the file to completed.
- Leave a final decision or progress note if the ending state is non-obvious.
