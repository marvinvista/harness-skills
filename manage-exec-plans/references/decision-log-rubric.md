# Decision Log Rubric

Add a decision entry when the team or agent chooses a path that future runs might question.

Good candidates:
- Architecture choices
- Tradeoffs accepted for launch
- Rejected alternatives that are likely to come up again
- Rollout or migration constraints
- Temporary compromises with an explicit cleanup path

Do not use the decision log for:
- Generic progress updates
- Repeating the milestone list
- Narrating every command that ran

Keep each entry short:
- What changed
- Why it changed
- What future work must respect
