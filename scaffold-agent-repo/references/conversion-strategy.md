# Conversion Strategy

When migrating an existing repo:

1. Add the `docs/` scaffold first.
2. Shorten `AGENTS.md` into a table of contents.
3. Move durable content out of giant root docs into indexed subdirectories.
4. Add execution-plan folders before starting the next large task.
5. Rebuild indexes and verify the scaffold.

Prefer incremental migration over a one-shot doc rewrite.
