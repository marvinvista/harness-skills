# Starter File Purpose

Use the starter files for these roles:

- `ARCHITECTURE.md`: top-level domain and layer map
- `docs/DESIGN.md`: durable design and UX principles
- `docs/PLANS.md`: planning system entrypoint
- `docs/PRODUCT_SENSE.md`: product heuristics and user value framing
- `docs/QUALITY_SCORE.md`: quality grading and gap tracking
- `docs/RELIABILITY.md`: runtime reliability expectations
- `docs/SECURITY.md`: security constraints and edge validation rules

Every file should either help navigation or hold stable knowledge.
