# Docs Layout Contract

Treat the repository as the system of record for agent-visible knowledge.

Prefer:
- One short entry file such as `AGENTS.md`
- A small number of stable top-level guides
- Indexed subdirectories for deeper material
- Checked-in plans, specs, and references

Avoid:
- One giant instruction file
- Deeply buried docs with no index path
- Duplicate sources of truth for the same workflow
- Drift between doc structure and code structure
